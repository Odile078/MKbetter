                                    Taylor Jones Ale3andr0s • 9 months ago • edited

                                    Check out my response above to 이경호. Could make use of this. However, to answer your exact question, you're probably better off making a parent <div> and loading it's HTML contents via JS or CSS. But either way, it will have just about the same performance impact (maybe a little less with JS) as just following the suggestion below:

                                    You could make use of Bootstrap's display utilities (d-none and d-{sm/md/lg/xl}-block) contained within two separate DIV's. Like so:


                                    //Only display on screens md and larger
                                    <div class="header d-none d-md-block">
                                    Insert Video Header Content Here
                                    </div>

                                    //Only display on screens md and larger
                                    <div class="header d-block d-md-none">
                                    Insert Slider Content Here
                                    </div>

                                    Another option would be to add the above "d-none d-md-block" classes to the <video> element and add your slider <div> below with the "d-block d-md-none" classes.

                                                                                                                                                      Start Bootstrap Mod leo • 4 months ago

                                                                                                                                                      The CSS of this snippet has a media query that opens a video on devices with a coarse pointer type, and no hover capabilities. This covers most touch screen devices. The reason for this is that a lot of mobile browsers don't support the autoplay functionality needed for this embedded HTML5 video element to play in the background, so the media query displays a static image on these devices instead. It all depends on how you want to use a media query, but the way to do it is by adding it into the CSS, and modifying from there.
